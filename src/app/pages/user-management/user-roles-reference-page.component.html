Roles & operation claims

How this fits together

  • Roles (JWT claim http://schemas.microsoft.com/ws/2008/06/identity/claims/role) are coarse labels such as Admin. They are issued at login and stored client-side for UI checks.
  • Operation claims are named permissions in the API (strings like users.read). They can be grouped and assigned to users; the Operation claims tab edits the catalog, and Assignments lists every user–claim row.
  • Per-user bulk assignment of operation claims uses Accounts → Permissions (replace all claims for that user in one save).

There is no separate “user roles” REST surface in this Angular client yet. If your backend exposes role assignment endpoints, wire them here and extend the Accounts detail view.
